# SEO Keyword Parser

Профессиональный парсер семантического ядра для российского рынка. Создал и собрал от нефиг делать @Mike_Hoover. Если пригодилось, проставьте пиво этому славному парню.

## Возможности

| Модуль | Что делает |
|--------|-----------|
| **Яндекс.Wordstat** | Selenium-парсинг, регионы, операторы `[!точная]` и `"фразовая"`, коэффициент спамности |
| **Google Trends** | Сезонность, динамика, растущие запросы, региональный интерес |
| **LSI-кластеризация** | TF-IDF и SERP-метод (Rush Analytics), определение интента |
| **Анализ конкурентов** | ТОП-10 SERP, Title/H1-H3/мета, CMS, Schema.org, скорость загрузки |
| **Serpstat API** | Частотность, CPC, конкуренция, ключи домена (при наличии ключа) |
| **Фильтр стоп-слов** | Встроенный словарь + свой список + паттерны |
| **Экспорт** | Excel (6 листов, условное форматирование) + CSV |

## Установка

```bash
# 1. Клонировать / распаковать проект
cd seo_parser

# 2. Создать виртуальное окружение
python -m venv venv
source venv/bin/activate        # Linux/macOS
# venv\Scripts\activate         # Windows

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Настроить конфиг
cp .env.example .env
# Открыть .env и заполнить своими данными
```

## Настройка

Отредактируйте `.env`:

```env
YANDEX_LOGIN=login@yandex.ru
YANDEX_PASSWORD=password
REGION_ID=54           # Екатеринбург
WORDSTAT_PAGES=5        # 5 страниц × 50 ключей = 250 ключей на запрос
SERP_TOP_N=10
CLUSTERS_N=20
HEADLESS=true
```

Отредактируйте `main.py` — укажите свои seed-ключи:

```python
SEED_KEYWORDS = [
    "купить диван",
    "диван в гостиную",
    "угловой диван",
]
```

## Запуск

```bash
python main.py
```

Результаты появятся в папке `output/`:
- `seo_keywords_YYYYMMDD_HHMM.xlsx` — полный Excel-отчёт
- `keywords_clean_YYYYMMDD_HHMM.csv` — очищенные ключи

## Советы

**Частотность и операторы:**
- Базовая частота (без операторов) — завышена в разы
- `[!точная]` — реальный спрос без словоформ
- Спамность = базовая / точная; если > 10 — ключ нежелателен

**Кластеризация:**
- SERP-метод точнее TF-IDF, но требует данные выдачи
- Порог 3 URL — стандарт Rush Analytics; при 2 — более мелкие кластеры

**Сезонность:**
- ratio > 4 = высокосезонный (публиковать статьи за 2-3 мес до пика)
- ratio < 1.5 = круглогодичный спрос

**Капча Яндекс:**
- При частом парсинге появляется капча. Решение: добавить паузы (`DELAY_MAX`)
- Сохраняйте cookies — позволяет не авторизоваться каждый раз
- Для промышленного сбора используйте API Key Tools (платно)
